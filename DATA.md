# Данные

Исходные данные не хранятся в Git: это уменьшает размер репозитория и исключает случайную публикацию файлов, распространяемых отдельно.

Для запуска ноутбука разместите файлы относительно корня репозитория:

```text
.
├── train_features.csv
├── train_targets.csv
└── DOTA 2/
    ├── train_matches.jsonl
    └── test_matches.jsonl
```

Ноутбук использует:

- `train_features.csv` — подготовленные табличные признаки матчей;
- `train_targets.csv` — идентификатор матча и целевой признак `radiant_win`;
- `train_matches.jsonl` — игровые логи для дополнительных признаков;
- `test_matches.jsonl` — тестовые игровые логи.

Идентификатор `match_id_hash` используется для контролируемого объединения источников. Перед запуском проверьте, что имена файлов и регистр каталога совпадают со структурой выше.
