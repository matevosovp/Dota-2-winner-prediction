# Dota 2: предсказание победы Radiant

[![Notebook checks](https://github.com/matevosovp/Dota-2-winner-prediction/actions/workflows/notebook-checks.yml/badge.svg)](https://github.com/matevosovp/Dota-2-winner-prediction/actions/workflows/notebook-checks.yml)

Проект бинарной классификации: по состоянию матча в фиксированный момент времени оценить вероятность победы команды **Radiant**.

> **Результат:** средний ROC-AUC на кросс-валидации — около **0.84**. Весь анализ, построение признаков и сравнение моделей находятся в [основном ноутбуке](Dota2winnerPrediction.ipynb).

## Что демонстрирует проект

- EDA и контроль качества табличных данных и JSONL-логов матчей;
- объединение источников по `match_id_hash`;
- feature engineering игровых сигналов, включая показатели преимущества команд и статистики героев;
- сравнение линейного baseline, Random Forest и CatBoost;
- стратифицированную кросс-валидацию с контролем утечек;
- подбор гиперпараметров CatBoost через Optuna и early stopping;
- интерпретацию важности признаков, ROC-кривые и анализ ошибок.

## Постановка задачи

- **Объект:** матч Dota 2 в фиксированный момент времени.
- **Таргет:** `radiant_win`.
- **Выход модели:** вероятность победы Radiant.
- **Основная метрика:** ROC-AUC — она оценивает качество ранжирования вероятностей независимо от выбранного порога классификации.

Случайное угадывание соответствует ROC-AUC `0.5`; итоговая модель достигает примерно `0.84`.

## Данные и признаки

Проект использует два типа источников:

1. подготовленные табличные признаки из `train_features.csv` и таргет из `train_targets.csv`;
2. JSONL-логи матчей для извлечения дополнительных временных и игровых сигналов.

Исходные файлы не публикуются в репозитории. Ожидаемая структура каталогов описана в [DATA.md](DATA.md).

## Эксперимент

Основные этапы ноутбука:

1. проверка схемы, пропусков, дубликатов и согласованности идентификаторов;
2. EDA по игровым характеристикам и балансу классов;
3. построение baseline-моделей;
4. извлечение признаков из логов и добавление агрегатов командного преимущества;
5. стратифицированная кросс-валидация;
6. настройка CatBoost с Optuna;
7. анализ feature importance, ROC-кривых и ошибок модели.

Для воспроизводимости используется `RANDOM_STATE = 42`, а версии библиотек зафиксированы в [requirements.txt](requirements.txt).

## Быстрый старт

Требуется Python 3.12.

```bash
git clone https://github.com/matevosovp/Dota-2-winner-prediction.git
cd Dota-2-winner-prediction

python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

jupyter lab Dota2winnerPrediction.ipynb
```

На Windows окружение активируется командой:

```powershell
.venv\Scripts\Activate.ps1
```

Перед запуском разместите исходные данные по инструкции из [DATA.md](DATA.md), затем выполните ячейки ноутбука последовательно сверху вниз.

## Ограничения

- результат является offline-оценкой и не измеряет качество вероятностей после реального deployment;
- исходный датасет распространяется отдельно, поэтому CI проверяет окружение и целостность ноутбука, но не переобучает модель;
- проект оформлен как исследовательский ноутбук, без отдельного production API.
