# Model card: Dota 2 Radiant win probability

## Назначение

Модель оценивает вероятность победы Radiant по snapshot матча. Основной сценарий — исследование ML-методологии на табличных игровых данных.

Не предназначена для ставок, финансовых решений, автоматического воздействия на игроков или использования вне исследовательского контекста.

## Вход и выход

- вход: признаки, доступные к единому зафиксированному моменту матча, плюс выбранные герои;
- target: `radiant_win`;
- выход: вероятность класса `1`;
- основная offline-метрика: ROC-AUC по внешним фолдам nested CV.

## Temporal boundary

Модель не должна видеть информацию, сформированную после snapshot. Запрещены итоговый урон, убийства Рошана, разрушенные башни, итоговый счёт и другие показатели завершённого матча из OpenDota JSON.

JSONL используется только для allow-listed pre-game metadata. На текущий момент это `hero_name`.

## Hero winrate

Winrate героя является target-derived признаком. Для train-строк применяется сглаженное leave-one-match-out значение. Для validation/test mapping строится только по соответствующей train-части. Неизвестный герой получает prior train-fold.

## Оценка

Исторический ROC-AUC около `0.84` отозван: он был получен с post-match признаками и не отражает заявленный сценарий. В репозитории намеренно нет новой цифры без повторного доступа к исходному датасету.

Протокол новой оценки:

1. внешний stratified split для измерения качества;
2. внутренний stratified split для выбора параметров;
3. единственный OOF-прогноз для каждой строки;
4. отчёт mean/std по внешним фолдам и общего OOF ROC-AUC.

CI проверяет этот протокол на синтетическом наборе, включая отсутствие self-target leakage.

## Риски и ограничения

- Dota 2 меняется между патчами, поэтому случайный split не измеряет temporal drift;
- ROC-AUC не гарантирует калибровку вероятностей;
- редкие и новые герои сильнее зависят от prior;
- неизвестна репрезентативность исходной выборки относительно всех регионов и уровней игроков;
- перед развёртыванием нужны time-based holdout, calibration curve и мониторинг drift.
